# 1) Criar um e-mail no Gmail e um perfil no Chrome

Antes de começar, crie um **e-mail no Gmail dedicado aos seus projetos** e use esse mesmo e-mail em um **perfil separado do Google Chrome**.

> Isso vai deixar seus projetos, logins, favoritos e histórico muito mais organizados ao longo da imersão.

## 1.1 Criar e organizar seu perfil no Chrome

1. Abra o Google Chrome
2. Clique na sua foto no canto superior direito
3. Clique em **Adicionar perfil**
4. Crie um perfil com o nome:

```text
Vibe Coding
```

5. Entre com seu Gmail de projetos

> A partir de agora, use esse perfil sempre que for acessar GitHub, Windsurf, Cursor e ferramentas da imersão.

---

## 1.2 Criar conta no GitHub

Acesse:

- https://github.com/signup

Passos:

1. Criar usuário
2. Confirmar e-mail
3. Escolher plano gratuito
4. Finalizar cadastro

> Recomendação: conecte o GitHub com o **mesmo Gmail que você criou para os projetos**.

# 2) Instalar o editor

## Opção principal recomendada: Windsurf

Antes de instalar o Cursor, vamos começar pelo **Windsurf**, porque ele tem uma interface baseada no VS Code e oferece uma experiência muito boa no plano gratuito para quem está começando no vibe coding.

Baixe o Windsurf:

- https://windsurf.com/editor

### Instalação e primeiros passos no Windsurf

1. Baixe e instale o Windsurf
2. Na primeira tela, clique em **Get Started**
3. Clique em **Sign up** para criar sua conta
4. Use sua conta do **Google** para entrar mais rápido
5. Se aparecer a oferta do plano pago, clique em **Skip for now**
6. Se aparecer a mensagem **"connection to server is erroring"**, feche no **X** e continue normalmente
7. Abra um projeto, ou seja, uma pasta do seu computador
8. Com o projeto aberto, abra o chat do **Cascade**, que é o chat de IA do Windsurf

> Para esta imersão, vamos indicar o **Windsurf** porque ele tem uma ótima experiência gratuita de IA e uma interface muito parecida com a do VS Code.

### Modelo recomendado no Windsurf

Dentro do chat do Cascade, selecione o modelo **SWE-1.6**.

> Esse modelo é uma excelente opção para começar e, no plano gratuito, costuma ser suficiente para os testes da imersão.

### 🧠 Quando usar cada um?

# 3) Instalar Git

O Git será responsável por salvar versões do projeto e conectar com o GitHub.

## 🪟 Windows

Baixe aqui:

- https://git-scm.com/download/win

Passos:

1. Baixar instalador
2. Avançar com **Next** em todas as etapas
3. Manter configurações padrão
4. Finalizar

Depois da instalação, abra o terminal dentro do Cursor:

1. Vá no menu superior **Terminal**
2. Clique em **New Terminal**
3. Com o terminal aberto, rode:

```bash
git --version
```

Se a versão aparecer na tela, significa que o Git foi instalado corretamente.

---

## 🍎 Mac

### Opção 1 — instalar via terminal

```bash
xcode-select --install
```

### Opção 2 — baixar manualmente

- https://git-scm.com/download/mac

Depois validar:

```bash
git --version
```

# 4) Primeira configuração do Git

Após instalar, rode os comandos no terminal **um por vez**:

```bash
git config --global user.name "Seu Nome"
```

Depois rode o próximo:

```bash
git config --global user.email "seuemail@exemplo.com"
```

> Esse nome e e-mail aparecerão nos seus commits sempre que você enviar um projeto para o GitHub.

# 6) Criar o primeiro repositório do projeto HTML e clonar na máquina

Nosso **primeiro projeto da imersão** será a criação do **currículo profissional do aluno em HTML**.

Por isso, já vamos iniciar com um repositório simples, visual e muito motivador.

> Fluxo da imersão: **GitHub → Clone → editar HTML → commit → push**

## 6.1 Criar o repositório no GitHub

1. Acesse o GitHub
2. Clique em **New repository**
3. Nome do projeto:

```text
curriculo-html-vibe-coding
```

4. Marque:

- ✅ Add a README file
- ✅ Add .gitignore

5. Clique em **Create repository**

> Esse será o primeiro projeto prático do aluno dentro da imersão.

---

## 6.2 Duas formas de começar o projeto na máquina

Antes de continuar, entenda um conceito importante: um **projeto é, na prática, uma pasta organizada no seu computador**.

A partir daqui, você pode seguir por **dois caminhos corretos** para começar, e conhecer os dois vai te ajudar a entender melhor como os projetos funcionam no mundo real.

### ✅ Caminho 1 — Clonar direto pelo Windsurf ou Cursor (mais fácil)

Na tela inicial do Windsurf ou do Cursor, clique em:

**Clone Repository**

Depois cole apenas a **URL do repositório** no campo solicitado:

```text
https://github.com/seuusuario/curriculo-html-vibe-coding.git
```

O editor vai baixar o conteúdo do projeto e abrir a pasta automaticamente para você.

> Esse é o caminho mais simples para iniciantes.

---

### 💻 Caminho 2 — Criar a pasta primeiro e depois conectar

Você também pode criar manualmente uma pasta no seu computador chamada:

```text
curriculo-html-vibe-coding
```

Depois, abra essa pasta no Windsurf, no Cursor ou no VS Code.

Com a pasta aberta, use o terminal para trazer o conteúdo do GitHub para dentro dela.

> Aqui você aprende um conceito importante: o repositório remoto pode alimentar a pasta local do seu projeto.

Depois disso, como a imersão será **100% vibe coding**, você já vai pedir a estrutura inicial para a IA.

### 🤖 Prompt para iniciar o projeto do currículo

Copie e cole no chat do Windsurf, do Cursor ou no Copilot Chat:

```text
crie uma pagina para eu apresentar o meu curriculo usando html, css, js puro e 100% responsivo. Use efeito, animações, icones e transições. A página deve ser moderna, bonita e profisisonal. White mode

A pagina deve ter as seguintes partes:
-Seção 1: Imagem, cargo e frase de impacto
•⁠  ⁠Habilidades
-Experiencia profissionais 
-Cursos e formacoes
-Idiomas
```text
Crie a estrutura inicial de um projeto HTML para um currículo profissional.

Quero a seguinte organização:
- pasta assets para imagem de perfil
- pasta css com arquivo style.css
- arquivo index.html
- arquivo README.md

Depois já gere um HTML inicial com nome, título profissional e uma área para experiência, formação e habilidades.
```

## 6.3 Fazer o primeiro commit e enviar para o GitHub

Depois de gerar e revisar seu projeto com a IA, faça o primeiro versionamento.

No terminal, rode:

```bash
git add .
git commit -m "feat: primeira versao do curriculo em html"
git push
```

### Conceitos básicos do GitHub

**Commit**

É o registro de uma alteração no seu projeto. Pense como uma foto salva do estado atual dos arquivos.

**Push**

É o envio dos commits da sua máquina para o GitHub.

**Pull**

É o caminho contrário: traz do GitHub para a sua máquina as atualizações mais recentes do projeto.

> Esses três conceitos vão aparecer o tempo todo no seu fluxo de vibe coding.
